# Plan 01: Cross-Platform Port Detection & Killing

**Phase:** 03-cross-platform-support  
**Goal:** Unified cross-platform port detection and process killing for Windows, macOS, and Linux

---

## Context

From `02-CONTEXT.md` and Phase 3 discussion:

- Project renamed to **PortMan** (npx portman)
- Single CLI must work on all platforms
- Port detection: Windows (`netstat`), macOS (`lsof`), Linux (`ss` or `netstat`)
- Process killing: `taskkill` (Windows), `kill -9` (Unix)
- System process detection: per-platform PID lists and process names
- npx support via package.json bin field

---

## Tasks

### Task 1: Cross-Platform Port Detector Library

**File:** `lib/port-detector.js`

**Implementation:**
- `getListeningPorts()` — platform-specific detection
  - Windows: `netstat -ano | findstr LISTENING`
  - macOS: `lsof -i -P -n | grep LISTEN`
  - Linux: `ss -tlnp` (fallback to `netstat -tlnp`)
- `getProcessName(pid)` — cross-platform process name lookup
  - Windows: `tasklist /FI "PID eq <pid>"`
  - macOS/Linux: `ps -p <pid> -o comm=`
- `killProcess(pid)` — cross-platform process killing
  - Windows: `taskkill /PID <pid> /F`
  - macOS/Linux: `kill -9 <pid>`
- `isSystemProcess(name, pid)` — per-platform system process detection
  - Windows: PIDs 0, 4 + known system executables
  - macOS: PIDs 0, 1 (kernel_task, launchd)
  - Linux: PIDs 0, 1, 2 (kernel, systemd/init)

**Success:** Library exports work correctly on all platforms
